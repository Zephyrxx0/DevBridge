## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user/chip data -> prompt builder | User text and chip metadata become backend prompt payload. |
| mention text -> regex replacement | User-controlled `@path` strings enter RegExp construction. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-33-01 | Tampering | `buildPromptContext` | mitigate | Use discriminated `ChatContextChip` union and clone artifacts; no optional `kind` fallback per D-08. |
| T-33-02 | Tampering | mention replacement | mitigate | Preserve regex escaping with `path.replace(/[.*+?^${}()|[\]\\]/g, "\\$&")`. |
| T-33-03 | Information Disclosure | mention handling | mitigate | Mentions remain labels only; helper accepts no file content for mentions and performs no fetch per D-04. |
</threat_model>
