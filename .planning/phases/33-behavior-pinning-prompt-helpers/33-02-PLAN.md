## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| PromptInput -> ChatInput | Local component emits text and optional files; app must consume text only. |
| ChatInput -> route page | Typed text payload crosses component boundary. |
| route page -> `/api/backend/chat/stream` | Expanded backendPrompt crosses client/backend API boundary. |
| route page -> existing file endpoint | Existing `/api/backend/repo/{repoId}/files/{path}` fetch remains only content fetch path. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-33-04 | Information Disclosure | `ChatInput` submit | mitigate | Destructure/use only `message.text`; do not read `message.files`; do not add attachment UI/actions per D-12. |
| T-33-05 | Tampering | chip drag/drop metadata | mitigate | Render/switch on discriminated `chip.kind`; malformed drop payloads must not flow as valid chips. |
| T-33-06 | Denial of Service | folder context expansion | mitigate | Preserve `slice(0, 8)` folder cap and `slice(0, 8000)` per-file cap. |
| T-33-07 | Elevation of Privilege | route submit refactor | accept | No auth/session endpoint changes; existing `/api/backend` rewrite and stream route behavior preserved. |
