# Phase 26: Admin Dashboard - Context

**Gathered:** 2026-05-16
**Status:** Ready for planning

<domain>
## Phase Boundary

This phase implements an Admin Dashboard to surface "intern confusion" topics based on developer query logs. It builds a UI that consumes the reports generated by the task scheduler (Phase 25) and utilizes the Fast Model (Gemma-4-9B-it) for summarization, without relying on advanced clustering techniques.
</domain>

<decisions>
## Implementation Decisions

### Data Scope
- **D-01: Per-Repository Filtering.** The "intern confusion" summaries will be filtered and displayed per-repository, rather than globally aggregated. This allows maintainers to clearly identify what developers struggle with in specific codebases.

### Dashboard UI
- **D-02: Markdown Feed.** The dashboard will render the generated Markdown reports directly as a simple feed/list. This allows for a fast implementation while clearly presenting the pre-computed AI summaries generated by the Phase 25 scheduler.

### Access Security
- **D-03: Supabase Roles.** The Admin Dashboard route will be protected using existing Supabase roles (e.g., checking for an `is_admin` flag on the user record in the database/auth module) to ensure only authorized maintainers and admins can access the insights.

### Pre-decided Patterns (from prior phases)
- **Report Generation:** Reports are pre-computed daily/weekly by the APScheduler job (decided in Phase 25).
- **Fast Model:** Uses `Gemma-4-9B-it` for the summarization capability (per Requirements FR-06).
</decisions>
