<objective>
Implement the storage schema for GitHub issues and the secure RPC mechanism for retrieving user OAuth tokens from Supabase, laying the groundwork for the issue mapping feature.

Purpose: To securely access user GitHub tokens without exposing `auth.identities` to the application directly (D-01), and to isolate issue vectors from code chunks (D-02).
Output: A new database migration, updated SQLAlchemy models, and refactored secret management.
</objective>
