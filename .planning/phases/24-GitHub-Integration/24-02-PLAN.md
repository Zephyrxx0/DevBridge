<objective>
Implement background issue synchronization using APScheduler and expose a new agent tool to ground LLM queries by associating GitHub issues with specific code chunks via pure pgvector cosine similarity.

Purpose: To continuously fetch and vectorize GitHub issues (D-03) and provide the AI agent with semantic context mapping issues to files (D-04) without incurring large token/context spikes by keeping the similarity search entirely inside the DB (D-05).
Output: APScheduler integrated into FastAPI's lifespan, and a LangGraph-compatible `map_issue_to_files` tool added to the Orchestrator.
</objective>

<task type="auto">
  <name>Task 2: Implement map_issue_to_files Agent Tool with On-Demand Sync</name>
  <files>api/agents/orchestrator.py</files>
  <action>Add a new `@tool` function `map_issue_to_files(repo_id: str, issue_number: int, limit: int = 5)`. The tool MUST first check if the issue exists in the `repo_github_issues` table; if it is missing, trigger an on-demand sync for that issue to satisfy D-03. Then, execute a pure Postgres SQL query joining `code_chunks` and `repo_github_issues` by cosine similarity (`ORDER BY code_chunks.embedding <=> repo_github_issues.embedding`). Return the top relevant files and snippets as a formatted string without fetching all vectors into Python memory. Add it to the `tools` list for the orchestrator.</action>
  <verify>
    <automated>grep -q "def map_issue_to_files" api/agents/orchestrator.py</automated>
  </verify>
  <done>Agent can invoke the tool to discover codebase files related to a specific GitHub issue number, syncing missing issues on-demand.</done>
</task>
