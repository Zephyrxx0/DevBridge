<task type="auto">
  <name>Task 2: E2E Data Cleanup Utility</name>
  <files>scripts/cleanup_e2e.py</files>
  <action>
    Create a Python script `scripts/cleanup_e2e.py` that:
    1. Deletes any code chunks in Supabase associated with the `e2e-test-repo`.
    2. Deletes raw file snapshots from GCS `gs://{bucket}/e2e-test-repo/`.
    3. Resets repository metadata in the `repositories` table.
    Use `google-cloud-storage` and `psycopg` (or existing DB session logic).
  </action>
  <verify>
    <automated>python scripts/cleanup_e2e.py --dry-run</automated>
  </verify>
  <done>Cleanup utility is ready to purge test artifacts after E2E runs.</done>
</task>

<task type="auto">
  <name>Task 3: Full Ingestion Loop E2E Test</name>
  <files>web/tests/ingestion_loop.spec.ts</files>
  <action>
    Implement `web/tests/ingestion_loop.spec.ts` to perform the following:
    1. **Setup**: Call `scripts/cleanup_e2e.py` to ensure a clean state.
    2. **Ingest**: Simulate a GitHub webhook or manually upload a sample file to GCS for the `e2e-test-repo`.
    3. **Poll**: Wait for the ingestion worker to complete (poll API or check DB state via helper).
    4. **Verify UI**: Navigate to `/repo/e2e-test-repo`, type a query related to the uploaded sample file.
    5. **Assert**: Verify that the AI response contains content from the sample file and cites the correct source.
    6. **Teardown**: Call cleanup again.
  </action>
  <verify>
    <automated>cd web && npx playwright test tests/ingestion_loop.spec.ts</automated>
  </verify>
  <done>The core RAG pipeline is verified from ingestion to retrieval via a real browser test.</done>
</task>
