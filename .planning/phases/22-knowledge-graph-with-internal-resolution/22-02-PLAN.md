       - Import `GraphBuilder` (from `api.ingestion.graph_builder`) and `GraphStoreManager` (from `api.db.graph_store`).
       - After the `code_chunks` ingestion is successfully completed and marked 'success':
         ```python
         # Build and save knowledge graph
         try:
             # repo_slug is available in _run_ingestion scope
             # initialize with repo_slug and engine (already passed to _run_ingestion)
             builder = GraphBuilder(repo_slug=repo_slug, engine=engine)
             graph_data = await builder.build_graph()
             
             store = GraphStoreManager(engine)
             await store.save_graph(repo_slug, graph_data)
             logger.info(f"Graph build for {repo_slug} completed.")
         except Exception as graph_err:
             logger.error(f"Graph build failed for {repo_slug} (non-fatal): {graph_err}")
         ```
       - Ensure this happens *after* the `ingestion_jobs` status update to 'success'.

    2. Run `graphify update .` to refresh the local knowledge graph after implementation.
